#### 关于ArrayList和LinkedList的相同
- 两者均不是线程安全的，在多线程情况下使用需要用Collection.syncronizedList方法进行包装

#### 关于两者的不同点
- ArrayList适合改查（不修改数组的大小），而LinkedList适合增删（不进行数组的遍历）
- ArrayList在添加/删除元素时需要动态扩容，因此效率不如ArrayList每个对象只要==增加或置空==节点中对象next和pre的双向指向，就可以完成数组的增删
- LinkedList在进行查询/修改元素时需要完整遍历一遍数组,通过first指针按顺序遍历查找，时间复杂度为O(n)

| 操作     | ArrayList         | LinkedList                            |
| ------ | ----------------- | ------------------------------------- |
| 增删     | O(n)              | O(1)                                  |
| 改查     | O(1)              | O(n)                                  |
| 内存占用大小 | 占用内存相对小，分配的空间利用紧凑 | 占用空间相对较高，需要存放各种节点和指针前后Node、first、last |
|        |                   |                                       |
